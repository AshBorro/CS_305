# CS 305 Portfolio Submission – Artemis Financial Security Project

## Artifact Included
Artemis Financial Practices for Secure Software Report

## Reflection

Artemis Financial was the client in this project. The company wanted to improve the security of its software application used for financial services. Because the company works with sensitive customer and financial data, it was important to make sure the application was secure, reliable, and protected from cyber threats. The main issue to address was identifying vulnerabilities in the software and implementing secure communication practices.

One thing I did well during this project was learning how to identify software vulnerabilities and understand how security weaknesses can impact a business. I used dependency-check tools to scan for known vulnerabilities in project dependencies and reviewed the code for areas that needed improvement. Coding securely is important because it helps protect customer data, prevents unauthorized access, and builds trust with users. Strong software security also helps reduce financial loss and damage to a company’s reputation.

One challenging part of the project was configuring secure certificates, HTTPS communication, and troubleshooting build errors while using Maven and dependency-check tools. Working through these challenges helped me better understand how security tools function in real-world development environments.

I increased layers of security by implementing secure HTTPS communication, using certificates, applying secure coding practices, and checking dependencies for vulnerabilities. In the future, I would continue using vulnerability scanners, code reviews, testing tools, and regular software updates to assess risks and choose the best mitigation techniques.

To make sure the software remained functional and secure after refactoring, I tested the application after each change and reran dependency scans to confirm vulnerabilities were reduced or removed. I also checked that the application still ran properly and that secure connections were working as expected.

Resources and tools I used during this project included Java, Eclipse IDE, Maven, Spring Boot, GitHub, OWASP Dependency-Check, and secure certificate generation tools. These resources will be helpful in future coursework and professional software development tasks.

This project demonstrates skills that would be valuable to future employers, including secure coding practices, vulnerability assessment, troubleshooting technical issues, using development tools, and improving software quality. It also shows that I can learn new technologies, solve problems, and apply cybersecurity concepts in software development.
